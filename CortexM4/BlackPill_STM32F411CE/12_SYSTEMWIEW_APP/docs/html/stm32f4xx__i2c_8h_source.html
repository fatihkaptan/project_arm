579/*
580 ===============================================================================
581 I2C State Monitoring Functions
582 ===============================================================================
583 This I2C driver provides three different ways for I2C state monitoring
584 depending on the application requirements and constraints:
585
586
587 1. Basic state monitoring (Using I2C_CheckEvent() function)
588 -----------------------------------------------------------
589 It compares the status registers (SR1 and SR2) content to a given event
590 (can be the combination of one or more flags).
591 It returns SUCCESS if the current status includes the given flags
592 and returns ERROR if one or more flags are missing in the current status.
593
594 - When to use
595 - This function is suitable for most applications as well as for startup
596 activity since the events are fully described in the product reference
597 manual (RM0090).
598 - It is also suitable for users who need to define their own events.
599
600 - Limitations
601 - If an error occurs (ie. error flags are set besides to the monitored
602 flags), the I2C_CheckEvent() function may return SUCCESS despite
603 the communication hold or corrupted real state.
604 In this case, it is advised to use error interrupts to monitor
605 the error events and handle them in the interrupt IRQ handler.
606
607 Note
608 For error management, it is advised to use the following functions:
609 - I2C_ITConfig() to configure and enable the error interrupts (I2C_IT_ERR).
610 - I2Cx_ER_IRQHandler() which is called when the error interrupt occurs.
611 Where x is the peripheral instance (I2C1, I2C2 ...)
612 - I2C_GetFlagStatus() or I2C_GetITStatus() to be called into the
613 I2Cx_ER_IRQHandler() function in order to determine which error occurred.
614 - I2C_ClearFlag() or I2C_ClearITPendingBit() and/or I2C_SoftwareResetCmd()
615 and/or I2C_GenerateStop() in order to clear the error flag and source
616 and return to correct communication status.
617
618
619 2. Advanced state monitoring (Using the function I2C_GetLastEvent())
620 --------------------------------------------------------------------
621 Using the function I2C_GetLastEvent() which returns the image of both status
622 registers in a single word (uint32_t) (Status Register 2 value is shifted left
623 by 16 bits and concatenated to Status Register 1).
624
625 - When to use
626 - This function is suitable for the same applications above but it
627 allows to overcome the mentioned limitation of I2C_GetFlagStatus()
628 function.
629 - The returned value could be compared to events already defined in
630 this file or to custom values defined by user.
631 This function is suitable when multiple flags are monitored at the
632 same time.
633 - At the opposite of I2C_CheckEvent() function, this function allows
634 user to choose when an event is accepted (when all events flags are
635 set and no other flags are set or just when the needed flags are set
636 like I2C_CheckEvent() function.
637
638 - Limitations
639 - User may need to define his own events.
640 - Same remark concerning the error management is applicable for this
641 function if user decides to check only regular communication flags
642 (and ignores error flags).
643
644
645 3. Flag-based state monitoring (Using the function I2C_GetFlagStatus())
646 -----------------------------------------------------------------------
647
648 Using the function I2C_GetFlagStatus() which simply returns the status of
649 one single flag (ie. I2C_FLAG_RXNE ...).
650
651 - When to use
652 - This function could be used for specific applications or in debug
653 phase.
654 - It is suitable when only one flag checking is needed (most I2C
655 events are monitored through multiple flags).
656 - Limitations:
657 - When calling this function, the Status register is accessed.
658 Some flags are cleared when the status register is accessed.
659 So checking the status of one Flag, may clear other ones.
660 - Function may need to be called twice or more in order to monitor
661 one single event.
662 */
663
