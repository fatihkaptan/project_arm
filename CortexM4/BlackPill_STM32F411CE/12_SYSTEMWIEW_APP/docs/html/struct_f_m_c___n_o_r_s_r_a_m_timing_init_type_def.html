◆ FMC_DataLatency

uint32_t FMC_NORSRAMTimingInitTypeDef::FMC_DataLatency

Defines the number of memory clock cycles to issue to the memory before getting the first data. The parameter value depends on the memory type as shown below:

  • It must be set to 0 in case of a CRAM
  • It is don't care in asynchronous NOR, SRAM or ROM accesses
  • It may assume a value between 0 and 15 in NOR Flash memories with synchronous burst mode enable
